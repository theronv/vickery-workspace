Git Workflow git
vd start [name]
Run at the start of every session. Pulls latest, shows new commits, warns about uncommitted work.
No argument → pulls the vd-cli repo (registry + retros). Pass a project name to pull a project repo.
vd start # pull vd-cli repo vd start chorify # pull chorify project repo
vd save [name]
Stage all changes, prompt for a commit message, commit, and push. No argument → saves vd-cli repo.
vd save # saves vd-cli (registry, retros, CLI source) vd save chorify # saves chorify project repo vd save --no-push # commit only, skip push
Prompt: What did you update? › type your message, hit enter
vd diff [name]
Show uncommitted changes.
vd diff vd diff chorify vd diff --staged # staged only
vd log [name]
Show recent commit history.
vd log # last 15 commits vd log chorify vd log -n 30 # last 30 commits
Project Management management
vd register [path]
Register a project once. Every command accepts the name instead of a path after this.
vd register ~/projects/chorify-pwa # prompts for: app name, bundle ID # stores in ~/vd-cli/projects.json # sync immediately: vd save # commit + push registry update
vd projects
Dashboard showing all registered apps and pipeline stage.
vd projects chorify ●●●○~○ Scaffold next mono-line ●●●●~● Complete
vd status <name>
Detailed pipeline view for one app — shows which artifacts exist and what to run next.
vd status chorify 1 ✅ Audit AUDIT.md found 2 ⬜ Prompts No PROMPTS.md Next: vd prompts chorify
Phase 1 — Build phase 1
vd audit <name>
The foundation command. Scans PWA source and produces two files every other command reads from. Answer the 3 context questions carefully — they feed into all content commands.
vd audit chorify vd audit chorify --skip-meta # skip context questions vd audit chorify --html # also generate HTML report
Writes: AUDIT.md · CLAUDE.md
vd opportunities <name>
35-check product scan across 6 categories: Stickiness, UX, Monetization, Social, Personalization, Best Practices.
vd opportunities chorify
Writes: OPPORTUNITIES.md
vd prompts <name>
Generates 6 personalized Claude Code prompts (P1–P6) tailored to this app's specific blockers, routes, and components.
vd prompts chorify
Writes: PROMPTS.md
vd scaffold <AppName>
Creates the full Expo + Hono monorepo with canonical VD stack pre-wired. Run from the parent directory.
cd ~/projects && vd scaffold Chorify vd scaffold Chorify --skip-env # fill API keys later
vd migrate <name> <dest>
Auto-converts PWA components to React Native: HTML→RN elements, onClick→onPress, localStorage→AsyncStorage, router→expo-router.
vd migrate chorify ~/projects/chorify vd migrate chorify ~/projects/chorify --dry-run
Writes to: dest/app-migrated/
vd retro <name>
Post-launch retrospective. Archived lessons automatically surface as warnings in future audits.
vd retro chorify vd retro chorify --analyze # infer from git history
Writes: RETRO.md → archived to ~/vd-cli/retrospectives/
Phase 2 — Ship phase 2
Requires: ANTHROPIC_API_KEY in your env (except vd brand). Add to ~/.zshrc: export ANTHROPIC_API_KEY=sk-ant-...
vd brand <name>
Extracts color palette + fonts from source. No API key required — pure static analysis. Classifies roles, computes WCAG contrast.
vd brand chorify
Writes: BRAND.md · brand-profile.json
vd listing <name>
Complete App Store listing. Templates for character-constrained fields; Claude API for description + keywords.
vd listing chorify
Writes: APP_STORE/listing.md
vd screenshots <name>
5 screenshot frame headlines + subheadlines + device spec sheet.
vd screenshots chorify vd screenshots chorify --frames 6
Writes: APP_STORE/screenshots.md
vd press <name>
Press kit with journalistic description, differentiator bullets, and founder quote (flagged for editing).
vd press chorify
Writes: MARKETING/press-kit.md
vd social <name>
X/Twitter thread, ProductHunt tagline + description, LinkedIn post, Instagram caption.
vd social chorify
Writes: MARKETING/social.md
vd help-articles <name>
Getting Started, FAQ (12 Q&As), per-screen how-tos (up to 4), Account & Billing. Import into Notion, Intercom, or Zendesk.
vd help-articles chorify
Writes: HELP/*.md
vd launch <name>
Runs all Phase 2 commands in sequence and organizes everything into LAUNCH_PACK/. Skip individual steps with flags.
vd launch chorify vd launch chorify --skip-brand --skip-help # App Store + marketing only
Writes: LAUNCH_PACK/ — BRAND.md · APP_STORE/ · MARKETING/ · HELP/
